# Client Suitability Assessment Standard Operating Procedure

## Document Information
- **Type**: Policy
- **Sensitivity**: Internal
- **Date**: 2026-01-01
- **Source**: WealthMgmt Operations & Compliance

---

## 1. Purpose

This SOP defines the standard procedure for assessing whether an investment recommendation is suitable for a given client. All RMs must follow this process for every recommendation made.

---

## 2. Suitability Assessment Steps

### Step 1: Verify Client Risk Profile is Current
- Check the date of the last risk questionnaire
- If more than 12 months old, initiate a re-assessment before proceeding
- If client circumstances have changed materially, re-assess immediately

### Step 2: Check Product Eligibility
- Confirm the product is on the approved product list
- Verify the product's risk classification
- Check minimum investment thresholds against client AUM

### Step 3: Cross-Reference Allocation Limits
Reference the applicable policy:
- Conservative Portfolio Policy for conservative clients
- Aggressive/Growth Suitability Rules for growth/aggressive clients
- Balanced Portfolio Guidelines for balanced clients

### Step 4: Assess Concentration Risk
- Review existing holdings before adding new positions
- Flag if any single asset class would exceed policy limits
- Flag if any single issuer would exceed 10% of portfolio

### Step 5: Document the Suitability Rationale
Write a suitability statement covering:
1. Why the product fits the client's risk profile
2. How it aligns with investment objectives
3. How it fits within allocation limits
4. Any risks disclosed to client

### Step 6: Escalate if Required
Refer to the escalation matrix in the General Compliance Framework for situations requiring licensed advisor involvement.

---

## 3. Suitability Verdict Categories

| Verdict | Meaning | Action |
|---|---|---|
| Suitable | Fully meets all suitability criteria | Proceed with recommendation |
| Marginal | Meets most criteria with caveats | Proceed with enhanced disclosure and documentation |
| Unsuitable | Fails one or more key criteria | Do NOT recommend; offer alternative |
| Requires Licensed Advice | Exceeds RM authorisation | Escalate to licensed advisor |

---

## 4. Common Suitability Failures (Red Flags)

- Recommending high-yield bonds to conservative clients
- Recommending technology sector concentrated funds to balanced clients
- Any alternative investment to conservative clients
- Crypto/digital assets to any client without licensed advisor
- Exceeding maximum allocation limits for any asset class
- Products with lock-up periods exceeding client's investment horizon
- Products where the minimum investment exceeds the allocation that is suitable

---

## 5. Documentation Requirements

Every suitability assessment must be recorded in the client file with:
- Date of assessment
- Product recommended
- Suitability verdict and reasoning
- Citations to policy sections relied upon
- Compliance status (cleared / needs review / blocked)
- RM signature and date
